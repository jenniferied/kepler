# Keplers digitale Identität: Vom Konzept zur Website

**15. März 2026**

Heute habe ich begonnen, Keplers digitale Identität systematisch aufzubauen. Die Ausgangslage: Ein Musiker mit Tracks auf Spotify, SoundCloud und YouTube, aber ohne kohärente Online-Präsenz. Die bestehende Website auf keplermusik.de ist ein einfaches WordPress-Setup mit Elementor — funktional, aber ohne eigene Persönlichkeit.

Die zentrale Frage: Wie übersetze ich die musikalische Identität von Kepler — elektronisch, atmosphärisch, zwischen Club und Melancholie — in eine visuelle und interaktive Web-Erfahrung? Nicht einfach eine weitere Linktree-Seite, sondern etwas, das die Musik widerspiegelt.

Erster Schritt war die Bestandsaufnahme: Welche Assets existieren bereits? Cover-Artworks der Trilogie-EPs (Red, Blue, Yellow Edition), die Singles "Liebe Schieben" und "Für mich", ein paar Musikvideos auf YouTube. Dazu das Logo — eine minimalistische Schrift auf schwarzem Grund.

## Die Technologie dahinter

Bewusste Entscheidung für Vanilla JavaScript mit ES6-Modulen statt eines Frameworks. Warum? Die Seite soll schnell laden, kein Build-Step im Weg stehen, und der Code soll nachvollziehbar bleiben. EventBus-Architektur für lose gekoppelte Komponenten. Tailwind CSS für das Grundgerüst, eigenes CSS für die Identität.

Die Ästhetik orientiert sich am Musikuniversum: Dunkler Hintergrund (#0a0a0a), monospacierte Schrift (IBM Plex Mono), sparsame Akzente. Kein Overdesign — die Musik soll sprechen.
